# Project Overview

## Project Name
AtomQuest Goal Portal

## Current Date
May 17, 2026

## Project Status
🚀 **Foundation In Progress**

## Purpose
Build a web-based internal portal that supports employee goal creation, manager approval, quarterly progress capture, and admin governance for the Atomberg Hackathon problem statement.

## Key Objectives
- [x] Define the architecture, roles, and data model for the MVP
- [x] Create the application foundation for role-based portal journeys
- [ ] Implement Phase 1 goal creation and validation workflow
- [ ] Implement Phase 2 achievement tracking and check-ins
- [ ] Deliver reporting, audit trail, and admin controls

## Technology Stack
### Backend
- `Next.js` route handlers for initial API delivery
- Repository abstraction for seeded in-memory data during the hackathon MVP
- Planned upgrade path to persistent storage in later phases

### Frontend
- `Next.js App Router`
- `TypeScript`
- Custom CSS with a dashboard-first layout

### Database
- `Phase 1`: seeded in-memory domain data
- `Phase 2+`: planned persistence layer with relational storage

### Tools & Services
- `Node.js`
- `npm`
- Vercel or equivalent hosting for demo deployment
- GitHub for source control and submission

## Execution Phases
- `Phase 0`: scope lock, architecture, data model, and screen map
- `Phase 1`: app shell, role-aware dashboards, and seeded domain data
- `Phase 2`: goal creation and employee submission flow
- `Phase 3`: manager approvals and rework loop
- `Phase 4`: quarterly check-ins and progress calculations
- `Phase 5`: reporting, governance, audit trail, and admin controls
- `Phase 6`: analytics, polish, and demo scripting
