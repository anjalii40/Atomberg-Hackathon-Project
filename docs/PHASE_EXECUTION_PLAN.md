# Phase Execution Plan

## Phase 0: Scope and Architecture
Exit criteria:

- roles and journeys are locked
- MVP stack is chosen
- entities and API contracts are defined

## Phase 1: Foundation
Exit criteria:

- app shell is running
- role switcher works
- employee, manager, and admin dashboards render from seeded data
- health and dashboard endpoints respond correctly

## Phase 2: Goal Creation
Exit criteria:

- employee can add, edit, delete, and submit goals
- validation rules enforce 100 percent total weight, 10 percent minimum, and max 8 goals

## Phase 3: Approval Workflow
Exit criteria:

- manager can review, approve, reject, and send back goals
- approved goals become locked

## Phase 4: Check-ins
Exit criteria:

- employees can capture planned vs actual achievement
- managers can add structured check-in comments

## Phase 5: Governance and Reporting
Exit criteria:

- admin can manage cycles and unlock exceptions
- exports and audit visibility are demo-ready

## Phase 6: Analytics and Demo Polish
Exit criteria:

- dashboards tell a clear story
- the 3 role journeys are stable and presentable

## Bonus Phase
Add only after core phases are stable:

- Entra ID SSO
- Teams and email notifications
- rule-based escalations
- advanced analytics and heatmaps
